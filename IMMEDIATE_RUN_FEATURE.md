# 立即运行任务功能说明

## 功能概述

在阴阳师自动脚本(OAS)的GUI调度器中，为每个任务添加了"立即运行"按钮，允许用户无视预定的调度时间直接运行任务，方便测试和调试。

## 功能特点

- **无视调度时间**: 点击"立即运行"后，任务会忽略原定的运行时间，立即加入执行队列
- **便于测试**: 开发者和用户可以快速测试特定任务的功能
- **保持原有配置**: 立即运行不会改变任务的原有调度配置
- **两个界面支持**: 在任务列表页面和概览页面都提供立即运行按钮

## 使用方法

### 在任务列表页面
1. 打开GUI程序
2. 进入"任务列表"页面
3. 找到要测试的任务
4. 点击任务时间显示旁边的蓝色"立即运行"按钮
5. 系统会显示成功消息，任务被设置为立即运行

### 在概览页面
1. 打开GUI程序
2. 在"概览"页面的"等待执行"区域
3. 找到要测试的任务
4. 点击任务时间显示旁边的"立即运行"按钮
5. 任务会被设置为立即运行

## 技术实现

### 后端实现

1. **ProcessManager类增强** (`module/gui/context/process_manager.py`)
   - 添加了 `gui_run_task_immediately()` 方法
   - 支持脚本运行时和停止时两种情况

2. **Script类增强** (`script.py`)
   - 添加了 `gui_run_task_immediately()` 方法
   - 调用Config的task_call方法设置立即执行

3. **ConfigModify类增强** (`module/config/config_modify.py`)
   - 添加了 `gui_run_task_immediately()` 方法
   - 处理脚本未运行时的立即执行请求

4. **Config类原有支持** (`module/config/config.py`)
   - 原有的 `task_call()` 方法已支持 `immediate=True` 参数
   - 通过 `_immediate_tasks` 集合跟踪立即执行的任务

### 前端实现

1. **TaskList组件** (`module/gui/qml/Content/TaskList.qml`)
   - 在每个任务项中添加立即运行按钮
   - 调用后端的 `gui_run_task_immediately()` 接口

2. **Taskmini组件** (`module/gui/qml/Component/Taskmini.qml`)
   - 在概览页面的任务小组件中添加立即运行按钮
   - 提供一致的用户体验

## 安全考虑

- 立即运行不会跳过任务的启用/禁用检查
- 如果任务被禁用，立即运行仍然会被跳过
- 立即执行标记是运行时的，不会保存到配置文件
- 错误处理完善，避免GUI崩溃

## 兼容性

- 与现有的调度系统完全兼容
- 不影响原有的任务调度逻辑
- 支持所有类型的任务立即运行

## 调试信息

立即运行功能会在日志中记录以下信息：
- GUI请求立即运行的任务名称
- 任务是否成功设置为立即执行
- 立即执行标记的设置和清除

## 故障排除

### 立即运行按钮无响应
1. 检查脚本进程是否正在运行
2. 查看控制台日志确认错误信息
3. 确认任务名称是否正确

### 任务未立即执行
1. 确认任务是否已启用
2. 检查是否有其他任务正在运行
3. 查看调度器配置是否正确

## 开发者注意事项

- 立即执行标记存储在 `config._immediate_tasks` 集合中
- 任务执行时应检查并清除立即执行标记
- 新增任务类型时要考虑立即执行的兼容性

## 已知限制

- **重启任务不支持**：重启任务是系统管理的特殊任务，不支持立即运行
- **任务必须已启用**：即使是立即运行，任务本身必须是启用状态
- **不影响调度时间**：立即运行后，任务的下次正常调度时间不会改变

3. 确认任务名称是否正确

1. 确认任务是否已启用

3. 查看调度器配置是否正确

### 中文任务名显示错误
- 系统会自动解析中文显示名到内部英文任务名
- 如果解析失败，检查翻译文件是否正确

## 技术细节

### 任务名称解析流程
1. 检查是否已经是下划线格式的内部名
2. 尝试将大驼峰格式转换为下划线格式
3. 反向查找翻译映射表，将显示名转换为内部名
4. 如果都失败，记录警告并返回失败

### 数据流
```
前端显示名(中文) -> 原始英文名(大驼峰) -> 内部名(下划线) -> 后端处理
```
